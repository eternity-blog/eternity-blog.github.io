词法分析设计原理与实现

实验内容

实验设计

实验思路设计

从左到右依次扫描字符，根据空格、回车、分隔符，将其划分为一个个子字符串，并对其进行分析分类，输出为一个个对应的二元组。

单词符号类别设计如下：

单词类别 单词自身值 类别编码
保留字 int、char、string、void、等等 1
标识符 除保留字外，以字母开头，字母数字组成的字符串 2
常数 无符号整数 3
运算符 +、-、*、/、++、>、<、&、|、!、等等 4
分隔符 ，、；、{、}、（、）、[、] 5

实验具体实现

结构体设计

该结构体中，设有字符数组 words，用来存储字符串；整型变量Label用来标记单词类别。

全局变量设计

  • 文件指针 f，指向待词法分析的文件。
  • 结构体数组 ptr[MAX]，存储所有的子字符串及其标签。
  • 字符变量 ch，表示当前从文件中提取出来的字符,采用全局变量避免函数之间相互传参。
  • 整型变量 num，作为结构体数组的索引。
  • 存有全部运算符的数组 operators[11]，用于判断当前读入字符是否是运算符。
  • 存有全部保留字的数组 keywords[25][10]，用于判断当前子字符串是否是保留字（关键字）。
  • 存有全部分界符的数组 delimiters[8]，用于判断当前读入字符是否为分界符。

函数设计

void ExtractText()

这是实现词法分析的主要函数，此函数中，根据每个子字符串的开头分别调用不同的函数进行进一步分析。

int illegal()

如果当前读入字符不是字母、数字、分界符、运算符中的任意一个，则判断为非法字符。

int ischar()

判断当前读入字符是否为字母，是则返回 1，否则返回 -1。

void Get_Words()

当ischar()函数返回1，则调用该函数，将字符依次读入，然后判断该子字符串是标识符还是关键字（注意标识符最长为32位，超过32位则取前32位）

int isnumber()

判断当前字符是否为数字，若是则返回1，否则返回-1。

int isdot()

判断当前字符是否为点，若是则返回1，否则返回-1。

void Get_Numbers()

当isnumber()函数返回1则调用该函数，进行无符号数进行判断。若读取过程中遇到字母，则表示错误变量名，标记错误为：“error:This is an illegal name.”。

若读取过程中遇到其他字符，则表示错误，标记错误为：“error:Encountered non defined input and not in comment.”。

int iskeywords()

如果当前子字符串是关键字则返回1，否则返回-1。

int isoperators()

如果当前字符为运算符则返回1，否则返回-1。

void Get_Operators()

如果函数isoperators()返回1则调用该函数，判断运算符是否合法。

如果连续运算符大于等于三个，则报错，标记错误为：“error: too many operators in a row.”。

如果运算符搭配错误，则报错，标记错误为：“error: wrong operators match.”。

同时当第一个运算符为 / 时，这里采用超前读字符，判断是否是单行注释或者是多行注释的开头。

合法运算符组合有：`+、++、+=、-、–、-=、*、**、*=、/、/=、%、%%、%=、=、==、>、>>、>=、<、<<、<=、!、!=、&、&&、&=、|、||、|=``

int isdelimiters()

若当前字符为分隔符则返回1，否则返回-1。

void Get_Delimiters()

如果函数isdelimiters()返回1，则调用该函数，将分隔符存入结构体数组。

LL(1)语法分析设计原理与实现

实验内容

实验设计

实验思路设计

  • 首先根据文法规则（G[S]），设计程序自动构造LL(1)分析表（包括设计程序自动求各非终结符号和终结符号的first集合和follow集合，并且根据first和follow集合构造LL(1)分析表）。
  • 读取词法分析程序输出作为输入。遍历将各字符放入token2数组。并用i代替全部标识符存入token数组，然后使用token进行LL(1)语法分析。

实验具体实现

E‘与T‘的宏定义

由于在生成first集和follow集合时，涉及到对单个字符的处理，所以采用宏定义将E’定义为e、T’定义为t，方便操作。

字符栈，用于存放分析过程中的文法符号。

文法规则G[S]产生式grammar_1 and grammar_2

首先定义一个二元组类型数据结构PSS（**pair<string, string>**）

然后将G[S]的每个产生式的左右部分别作为PSS的第一个元素first和second，并存入vector数组。

由于有些产生式可能对应两个右部，所以两个vector数组。

FIRST集

  • 将每个非终结符号作为键，将其first集作为值，存入数据结构unordered_map中。
  • 将每个终结符号作为键，将其first集作为值，存入数据结构unordered_map中
  • 将每个产生式右部复合符号串作为键，将其first集作为值，存入数据结构unordered_map中

FOLLOW集

将每个非终结符号作为键，将其follow集作为值，存入数据结构unordered_map中

LL(1)分析表Table

  • 首先定义一个二元组数据结构PCC，其中PCC.first对应LL(1)分析表中的纵列的非终结符号，PCC.second对应LL(1)分析表中横行的终结符号
  • 然后自定义哈希函数hash_pair，并将其作为模板参数传递给unordered_map。这样可以确保对于pair类型的键，能够正确计算哈希值

  • 首先根据规则生成非终结符号的FIRSTVT集和LASTVT集

具体函数设计

void Solve_FIRSTVT()

  • 首先初始化：

    • 若有规则 U→b…, 或 U→Vb…,则 b ∈ FIRSTVT(U)
    • 然后置（U，b）为真，并将该二元组压入栈F

  • 然后循环所有文法产生式：

    • 弹出栈顶元素，记（V,b）
    • 对每个形如U->V…的规则
    • 若F（U，b）为假，变为真，进STACK栈
    • 若F（U，b）为真，再循环

  • 结果：

    • FIRSTVT(U)={b∣F(U,b)=TRUE}

void Print_FIRSTVT()

void Solve_LASTVT()

思路与FIRSTVT集求解类似

  • 首先初始化：

    • 若有规则 U→…b, 或 U→…bV,则 b ∈ FIRSTVT(U)
    • 然后置（U，b）为真，并将该二元组压入栈F

  • 然后循环所有文法产生式：

    • 弹出栈顶元素，记（V,b）
    • 对每个形如U->V…的规则
    • 若F（U，b）为假，变为真，进STACK栈
    • 若F（U，b）为真，再循环

  • 结果：

    • LASTVT(U)={b∣F(U,b)=TRUE}

void Make_Table()

遍历所有文法产生式规则：

  • IF Xi 和 Xi+1 均∈Vt， THEN 置 Xi = Xi+1

  • IF i <= n-2 且 Xi 和 Xi+2 均∈Vt，但 Xi+1 ∈Vn，THEN 置 Xi = Xi+2

  • IF Xi ∈Vt， 但 Xi+1 ∈ Vn THEN:

    • FOR b ∈ FIRSTVT(Xi+1) Do 置 Xi < b

  • IF Xi ∈ Vn， 但 Xi+1 ∈ Vt THEN:

    • FOR a ∈ LASTVT(Xi) Do 置 a > Xi+1

  • 拓广文法：G[E]: #E#

    • # < a（a ∈ FIRSTVT(E)）
    • a > *#*（a ∈ LASTVT(E)）
