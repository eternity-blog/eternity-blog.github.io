map

优点

内部用红黑树实现，内部元素具有有序性，查询删除等操作时间复杂度均为O(logN)

缺点

占用空间，红黑树里每个结点需要保存父子节点和红黑性质等信息，空间占用较大

unordered_map

优点

内部用哈希表实现，查找速度非常快（适用于大量的查询操作）

缺点

建立哈希表比较耗时

priority_queue

头文件

默认是大根堆

优先队列是在正常队列的基础上加了优先级，保证每次的队首元素都是优先级最大的。

每次操作队列中的元素都是按优先级排序的

声明

priority_queue pq

方法函数

设置优先级

基本数据类型的优先级设置

priority_queue<int, vector<int>, less<int> > pq;
//最后两个>之间要有空格

第二个参数：
vector< int > 是用来承载底层数据结构堆的容器，若优先队列中存放的是double型数据，就要填vector< double >
总之存的是什么类型的数据，就相应的填写对应类型。

第三个参数：
less< int > 表示数字大的优先级大，堆顶为最大的数字（大根堆）
greater< int >表示数字小的优先级大，堆顶为最小的数字（小根堆）
int代表的是数据类型，也要填对应的类型

结构体（或自定义）优先级设置

优先级设置可以定义在结构体内进行小于号重载，也可以定义在结构体外

//要排序的结构体（存储在优先队列里面的）

STL常用函数

accumulate(beg,end,init)

时间复杂度：O（N）

作用：对一个序列的元素求和

beg为序列的初始地址，end为序列的尾地址，init 为对序列元素求和的初始值

返回值类型：与init

atoi(const char*)

作用：将字符串转换为int类型

atoi不做范围检查，如果超出上界，输出上界，超出下界，输出下界。

int范围：-2147483648~2147483647

stoi(const string*)

作用：将对应string类型字符串转换为数字

stoi会做范围检查，默认必须在int范围内，如果超出范围，会出现RE（Runtime Error）错误。

to_string()

作用：将数字转化为字符串,支持小数（double）

is_sorted(beg,end)

时间复杂度：O（N）

作用：判断序列是否有序(升序)，返回bool值

lower_bound() 和 upper_bound()

时间复杂度：O（logN）

作用：二分查找

max_element() 和 min_element()

时间复杂度：O（N）

作用：找最大值最小值

max() 和 min()

时间复杂度：O（1）

作用：找多个元素的最大值和最小值

reverse(beg,end)

时间复杂度：O（N）

作用：对序列进行翻转

sort() 和 stable_sort()

时间复杂度：O（NlogN）

作用：对一个序列进行排序

stable_sort()功能和sort()基本一样

区别在于stable_sort()能够保证相等元素的相对位置，排序时不会改变相等元素的相对位置

unique(beg,end)

时间复杂度：O（N）

作用：消除重复元素，返回消除完重复元素的下一个位置的地址

如：a[] = {1,2,3,3,4 };

unique之后a数组为{1,2,3,4,3}前面为无重复元素的数组，后面则是重复元素移到后面，返回a[4]位置的地址（不重复元素的尾后地址）

消除重复元素一般需要原序列是有序序列

运用：离散化

_gcd(a,b)

作用：求a和b的最大公约数

_lg(a)

作用：求一个数二进制下最高位位于第几位(从第0位开始)（或二进制数下有几位）
