优化思路

动态规划的优化一般都是对代码或者集合最值方程进行一个等价变形。在考虑动态规划问题的时候，一定要先把基本的形式写出来，然后再对它进行优化。

首先，根据优化前的起码，f[i][j] 是从上到下，一行一行这样填满的：

看一下 f[i][j] 的计算公式：f[i][j] = max(A, B)。

只用到了f[i - 1][j]，f[i-1][j - vi]，即只用到了 f[i - 1] 这一层，并且用到的体积为 j 和 j - vi，都是小于等于 j 的。

因此可以从体积为 V 开始，利用f[i - 1]的数据，求解出 f[i][j]，把 f[i][j] 放到 f[i -1][j] 的位置上。这样 f 数组就能优化到一维了。

并且，当 背包容量小于 vj 的时候，f[i][j] = max{f[i - 1][j]，0} = f[i - 1][j]。所以 j 只需要从 V 遍历到 vj 即可。

极限尺度上是一样的

由上述两式，可以得到递推关系： f[i][j] = max(f[i][j - v] + w，f[i - 1][j])

思路分析

思路一

闫氏DP分析法：
状态表示：f(i，j) 集合：从 (1，1) 到 (i，j) 所有方案的集合
属性：Max

状态计算：这个点可以由 (i−1，j) 走过来，即f(i−1，j) + a(i，j)
这个点也可以由 (i−1，j−1) 走过来，即f(i−1，j−1) + a(i，j)
所以状态转移方程就是 f(i，j) =max{f(i−1，j)，f(i−1，j−1)} + a(i，j)

答案：根据定义，这里答案就是最下面那一层的和，即max1 ≤ i ≤ n{f(n，i)}

思路二

闫氏DP分析法：
状态表示：f(i，j) ：从 (i，j) 到最后一层的所有方案
属性：Max

状态计算：这条路径可以从(i+1，j)走上来，即f(i+1，j) + a(i，j)
也可以从 (i+1，j+1) 走上来，即 f(i+1，j+1) + a(i，j)
所以状态转移方程就是f(i，j) = max{f(i+1，j)，f(i+1，j+1)} + a(i，j)

答案：根据定义，答案就是 f(1，1)

最短编辑距离

题目描述

思路分析

状态表示

集合：g[i][j]表示 a[1i] 变成 b[1j] 的操作集合

属性：Min

状态计算

以对 a 中的第i个字母操作不同进行状态划分

若是删除操作，则保证a[1i-1] 与 b[1j] 相同，即 f[i - 1][j] + 1

若是增加操作，则保证a[1i] 与 b[1j-1] 相同，即 f[i][j - 1] + 1

如果是替换操作，需考虑 a[i] 是否等于 b[j]，即 f[i - 1][j - 1] + (a[i] != b[j])

上述三者取 Min 即可

最短Hamilton路径

题目描述

算法分析

时间复杂度：O(2nn2)

Q1：f[i][j] 中 i 和 j 具体表示什么?

A1：i 为一个二进制数，每一位分别表示是否经过该位上的点；j表示从0号点开始，最后在j号点落脚的最短路径。

Q2：为什么更新状态转移方程时需要将 i 减去 2j ？

A2：i的每一位分别表示是否经过该位上的点。

f[i][j]=min(f[i][j]，f[i−2j][k] + w[k][j])，在调用 f[i−2j][k] 时，这段路径的落脚点为k点，不一定经过j点，所以需要把j位改成0（因为i表示的二进制数中f[i][j]这段路径经过了j，第j位为1）

Q3：最后的答案为什么存在f[2n − 1][n − 1]中？

A3：i = 2n - 1表示有n个1，也就是所有的点都走完了，j = n - 1，表示最后落脚于 n - 1号点。

算法分析

本来是一个dfs的过程，遍历所有的位置，找到从当前位置往下走的最大路径，再取最大值，可是这样做会有很多重复的位置被重新计算过，因此可以利用空间换时间的思想，把遍历过的位置往下走的路径的最大值进行记录，这就是记忆化搜索

时间复杂度：O(n2)
